## What is APB Protocol?

Advanced Peripheral Bus (APB) is a part of the AMBA (Advanced Microcontroller Bus Architecture) family developed by :contentReference[oaicite:0]{index=0}.

APB is designed for communication with low-bandwidth peripherals that do not require high-performance data transfers. It provides a simple and low-power interface for connecting peripherals such as UART, GPIO, Timers, SPI, and I2C modules.

Unlike high-speed buses, APB uses a simple state-based transfer mechanism which reduces power consumption and hardware complexity.

APB supports:
- Read operations
- Write operations
- Simple synchronous communication
- Low power consumption
- Easy implementation using FSM

Because of its simple architecture, APB is widely used in SoC (System on Chip) designs for peripheral interfacing.

The APB protocol mainly operates using three phases:
1. Idle phase
2. Setup Phase
3. Access Phase

## Important APB Signals

### PCLK
PCLK is the clock signal used to synchronize all APB operations.

### PRESETn
PRESETn is an active-low reset signal used to initialize the APB system.

### PSEL
PSEL is asserted by the master to select a particular slave device.

### PENABLE
PENABLE indicates the Access phase of the APB transfer.
- LOW → Setup phase
- HIGH → Access phase

### PWRITE
PWRITE determines the type of transfer:
- HIGH → Write operation
- LOW → Read operation

### PREADY
PREADY is generated by the slave.
- HIGH → Transfer completed
- LOW → Wait state inserted

### PSLVERR
PSLVERR indicates an error condition during transfer.
